## 1. デザインコンセプト

**テーマ**: "Magazine Pop" (マガジン・ポップ)
**キーワード**: 雑誌風、エディトリアル、清潔感、遊び心、余白

**概要**:
従来の管理画面的なUIから脱却し、クリエイターのポートフォリオ自体が一つの「雑誌」のように見えるデザインを目指す。
真っ白なキャンバスに、鮮やかなアクセントカラーと大胆な文字組みを配置し、コンテンツ（作品）を際立たせる。

## 4. タイポグラフィ

### 4.1 英数字（Headings / Numbers）

- **Font**: `Outfit` (Google Fonts)
- **特徴**: 幾何学的でモダンなサンセリフ体。数字が美しく、見出しに使うとロゴのような強さが出る。
- **使用箇所**: ページタイトル、セクション番号 ("01", "02")、統計数字。

### 4.2 日本語（Body / UI Text）

- **Font**: `Noto Sans JP` (Google Fonts)
- **特徴**: 癖がなく読みやすい。ウェイト(太さ)の使い分けで階層を作る。
- **使用箇所**: 本文、メニュー、ボタンのラベル。

## 5. UIコンポーネント定義

### 5.1 カード (Cards)

- **背景**: 白 (`#FFFFFF`)
- **枠線**: 1px solid `#E5E7EB` (または黒 `#1A1A1A` で強調する場合あり)
- **影**: なし (Flat) または 非常にシャープな影 (`box-shadow: 4px 4px 0px #1A1A1A`)
- **角丸**: `rounded-md` (やや小さめの角丸)

### 5.2 ボタン (Buttons)

| バリエーション | スタイル |
|--------------|---------|
| Primary | 黒背景 + 白文字 + 角丸。ホバーで少し浮く。 |
| Secondary | 白背景 + 黒枠線 + 黒文字。 |
| Accent | アクセントカラー（Green/Coral）背景 + 黒文字。 |

### 5.3 装飾要素 (Decorations)

- **Squiggly Lines (波線)**: セクションの区切りに使用。
- **Dots (ドットパターン)**: 背景の一部や画像の装飾に使用。
- **Bento Grid**: 異なるサイズの四角形を組み合わせたグリッドレイアウト。

## 6. 画面レイアウト設計

### 6.1 公開プロフィール画面 (C01)

雑誌の「特集ページ」のようなレイアウト。

#### ヘッダーエリア（上部）

- **名前**: 特大サイズで表示 (例: "KANA.T")。独立したタイポグラフィとして配置。
- **肩書き**: 名前の上に小さく欧文で (例: "UI/UX DESIGNER")。
- **アバター**: 角丸正方形、または有機的な形状で切り抜き。背景にドットパターンを敷く。

#### スキル・ステータス（Bento Block）

- **Status Block**: 現在の状況 ("お仕事募集中"など) を吹き出し風に表示。
- **Skills Block**: ツールアイコン + ドットバーで習熟度を表示。
  - `Photoshop ●●●●○`
  - `Figma ●●●●●`

#### 作品グリッド（下部）

- **見出し**: 大きな数字 "02 WORKS" を左端に配置。
- **レイアウト**: 縦横比の異なる画像を隙間なく並べる (Masonry)。
- **ホバー**: 画像にカーソルを合わせると、白い枠とタイトルが浮かび上がる。

### 8.2 余白・アイコンサイズの改善

現状の余白やアイコンサイズにインパクトがないため、以下の改善を行う。

#### 8.2.1 余白（Spacing）の改善

| 箇所 | 現状 | 改善後 | 効果 |
|-----|------|--------|------|
| セクション間 | `py-8` (32px) | `py-16` (64px) | 呼吸感のある雑誌レイアウト |
| カード内パディング | `p-4` (16px) | `p-6` (24px) | コンテンツの重要度を強調 |
| ヘッダーエリア高さ | 固定高 | `min-h-[40vh]` | ヒーローセクション感を演出 |
| グリッドギャップ | `gap-4` | `gap-6` | 作品の独立性を高める |
| モバイル余白 | `px-4` | `px-6` | エッジからの余裕を確保 |

#### 8.2.2 アイコン・アバターサイズの改善（イラストレーター特化）

イラストレーターの個性を強調するため、アバターやアイコンを「特大サイズ」に変更する。

| 箇所 | 現状 | 改善後 | 備考 |
|-----|------|--------|------|
| **メインアバター** | `160px` | `240px` (PC) / `180px` (SP) | **インパクト重視**。作品の一部として扱うレベルの大きさ。 |
| **SNSアイコン** | `24px` | `48px` or `56px` | クリックしやすく、各プラットフォームへの導線を太くする。 |
| スキルアイコン | `20px` | `40px` | ツール技術（Ps, Ai, Figma等）の視認性を向上。 |
| ナビゲーション | `20px` | `32px` | メニューの押しやすさ改善。 |

※ アバターは単なる正方形ではなく、`rounded-3xl` (角丸強め) や `Squircle` (スーパー楕円) を採用し、柔らかい印象を与える。

### 8.3 作品の見せ方改善

作品の魅力を最大限に引き出すためのデザイン改善。詳細画面に遷移しなくても魅力が伝わるようにする。

#### 8.3.1 作品カードの改善

**Concept**: "Immersive Thumbnail" (没入型サムネイル)

```
┌─────────────────────────────────┐
│                                 │
│                                 │
│           [作品画像]             │
│        border-radius: 12px      │
│        hover: scale-105         │
│        transition-all           │
│                                 │
│                                 │
└─────────────────────────────────┘
  Title (text-xl, font-bold)
  YYYY.MM.DD  Category
```

- **画像サイズ**: PC版では3カラム→2カラムに変更し、**1枚あたりの表示サイズを1.5倍**にする。
- **角丸**: `rounded-xl` (12px〜16px) で親しみやすさを出す。
- **ホバー**: 画像自体を少し拡大 (`scale-105`) し、シャドウを濃くすることで「浮き上がる」感覚を作る。

#### 8.3.2 ギャラリーレイアウトの改善

| 機能 | 説明 |

| **Large Masonry** | カラム数を減らし（PC: 2列, SP: 1列）、画像を大きく見せるMasonryレイアウト。 |

| **Lazy Motion** | スクロールに合わせてふわっと浮き上がる出現アニメーション (`framer-motion` like)。 |

## 9. Phase 6: 全ページデザイン統一 — Framer Motion + ネオブルータリズム（v5.3〜v5.3.3）

**追加日**: 2026-02-10

LP（ランディングページ）で確立した「Magazine Pop」デザインを認証ページ・ダッシュボード全体に適用し、サービス全体のデザイン品質を統一した。

### 9.1 共通基盤コンポーネント（3ファイル新規作成）

| コンポーネント | ファイル | 説明 |
|--------------|---------|------|
| アニメーションバリアント | `lib/motion.ts` | LP用・ダッシュボード用の全バリアントを集約 |
| PageHeader | `components/ui/page-header.tsx` | アイコン + タイトル + SquigglyLine + アクションボタン |
| FloatingShapes | `components/ui/floating-shapes.tsx` | 背景フローティング幾何学図形（auth用5個 / dashboard用3個） |

### 9.2 アニメーション設計

#### バリアント一覧（`lib/motion.ts`）

| バリアント名 | 対象 | duration | 移動量 | 備考 |
|-------------|------|----------|--------|------|
| fadeInUp | LP | 0.5〜0.6s | y: 30px | whileInView once: true |
| staggerContainer / staggerItem | LP リスト | 0.1s間隔 | — | |
| dashFadeIn | ダッシュボード | 0.3s | y: 12px | 軽量・高速 |
| dashStaggerContainer / dashStaggerItem | ダッシュボード リスト | 0.06s間隔 | y: 12px | |
| scaleIn | 空状態アイコン | 0.4s | scale: 0→1 | spring物理 |
| neoBrutalistHover | カードホバー | — | y: -4px | shadow追加 |
| cardHoverLift | 軽量リフト | — | y: -2px | shadow追加 |

#### 定数

```typescript
NEO_CARD = "border-2 border-slate-900 shadow-[4px_4px_0px_#1A1A1A] rounded-lg bg-white"
NEO_CARD_ACCENT = "border-2 border-slate-900 shadow-[6px_6px_0px_#1A1A1A] rounded-lg bg-white"
```

#### パフォーマンス方針

- ダッシュボード: 軽量アニメーション（0.25〜0.3s）。LPより高速
- パララックスはLP限定。ダッシュボード/認証ではCSSフロートのみ
- `whileInView` は `once: true` で再トリガー防止
- モバイル: FloatingShapes dashboard版は `hidden md:block`

### 9.3 認証ページ（5画面）

対象: login, signup, forgot-password, reset-password, verify-email

#### 共通変更

| 要素 | 変更前 | 変更後 |
|-----|--------|--------|
| 背景 | `bg-slate-50` | 白背景 + FloatingShapes（auth, 5図形） |
| カード | 薄い枠線 | `border-2 border-slate-900 shadow-[6px_6px_0px_#1A1A1A]` |
| ロゴ | なし | PicMeロゴ + SquigglyLine |
| タイトル | 通常フォント | `font-outfit font-black` |
| エントランス | なし | Framer Motion fadeInUp |
| エラー/成功 | 静的表示 | AnimatePresence + スライドアニメーション |

### 9.4 ダッシュボード共通（2ファイル）

#### layout.tsx

```
┌──────────────────────────────────────────────────┐
│ [PicMe ロゴ]                        [≡ Menu]    │ ← モバイルヘッダー (md:hidden)
├───────────┬──────────────────────────────────────┤
│           │                                      │
│ Sidebar   │ PageHeader + コンテンツ               │
│ (desktop) │                                      │
│ md:flex   │ FloatingShapes (dashboard, 3個)       │
│           │                                      │
├───────────┴──────────────────────────────────────┤

└──────────────────────────────────────────────────┘
```

- モバイル: ハンバーガーメニュー → AnimatePresence + motion.div スライドイン（左から） + オーバーレイ背景
- usePathname() でページ遷移時にモバイルメニュー自動クローズ
- ローディング状態: PicMeロゴ + SquigglyLine + スピナー

#### sidebar.tsx

| 要素 | 変更内容 |
|-----|---------|
| variant prop | `'desktop'` / `'mobile'` 対応（モバイルはフルスクリーンオーバーレイ内） |
| ロゴ | PicMeロゴ横に SquigglyLine |
| ナビアイテム | `whileHover={{ x: 4 }}` 微シフト |
| PRO機能 | PRO バッジ表示 |
| 公開ページリンク | ネオブルータリスト border + brand-green背景 + hover shadow |
| モバイル | X 閉じるボタン、メニュー項目クリック時に自動クローズ |
