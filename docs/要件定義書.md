## 1. プロジェクト概要

### 1.1 プロジェクト名
**PicMe（ぴくみー）**

### 1.2 プロジェクトの目的
イラストレーターが簡単に自分のポートフォリオサイトを作成できるWebサービスを提供する。プログラミングスキルがなくても、魅力的で個性的なプロフィールページを誰でも作成できる。

### 1.3 コンセプト
- **Twitterプロフィールのビジュアル拡張版**
- ノーコードで誰でも簡単に作成可能
- 作品の魅力を最大限に引き出す可愛くてポップなデザイン
- SNSとシームレスに連携

### 1.4 プロジェクトスローガン
「あなたの作品を、もっと輝かせよう」

## 3. 主要機能

### 3.1 MVP機能（Phase 1 - 必須）

#### 3.1.1 ユーザー認証・ログイン

- レスポンシブデザイン（PC/タブレット/スマートフォン対応）
- OGPメタタグの自動生成

- カラーテーマの変更（Pro以上）

### 3.3 Phase 3機能（有料機能）

## 4. 非機能要件

### 4.1 対象外機能（初期バージョン）
- コミッション受付の高度な管理
- 作品販売・EC機能
- オンライン決済（Stripe課金を除く）
- メッセージング・チャット機能
- コミュニティ・フォロー機能

### 4.2 パフォーマンス要件
- ページ読み込み速度: 3秒以内
- 画像の自動最適化・圧縮（Cloudinary）
- レスポンシブデザイン（モバイル対応必須）
- 同時接続数: 1,000人以上対応

### 4.3 セキュリティ要件
- HTTPS通信（SSL/TLS必須）
- パスワードのBCryptハッシュ化
- CSRF対策（Spring Security）
- XSS対策（入力のサニタイズ）
- SQLインジェクション対策（JPA Prepared Statement）
- ファイルアップロード検証
  - MIME typeチェック
  - ファイルサイズ制限
  - 許可拡張子: jpg, png, gif, webp

### 4.4 可用性要件
- 稼働率: 99.9%以上
- 定期メンテナンス: 月1回、深夜帯
- バックアップ: 毎日自動実行

### 4.5 スケーラビリティ要件
- 10,000ユーザーまで対応（Phase 1目標）
- 水平スケーリング可能な設計
- CDN利用による配信最適化

### 4.6 開発・運用管理

#### サーバーの再起動方法（Docker）

開発環境でのトラブルシューティングや設定反映のために、以下のコマンドでコンテナを再起動できます。

- **全サービス再起動**: `docker-compose restart`
- **フロントエンドのみ**: `docker-compose restart frontend`
- **バックエンドのみ**: `docker-compose restart backend`

※ 完全にクリーンな状態で再起動したい場合は `docker-compose down && docker-compose up -d` を実行してください。

## 8. 競合との差別化

### 8.1 主要競合
- **Linktree / lit.link**: SNSリンク集約サービス
- **Carrd**: シンプルなページビルダー
- **Portfoliobox**: ポートフォリオ特化型

### 8.2 PicMeの独自性

#### イラストレーター特化
- 作品ギャラリーが充実（Linktreeより圧倒的に強い）
- イラスト寄りのポップなデザイン
- カテゴリー・タグ機能で作品整理

#### 低価格
- Pro: ¥680/月（Portfoliobox $10/月より安い）
- 無料プランでも画像10枚（Linktree無料版は1枚のみ）

#### 簡単操作
- ノーコードで5分で作成可能
- リアルタイムプレビュー
- ドラッグ&ドロップ操作

#### 日本市場特化
- 日本語対応
- 日本のイラストレーターコミュニティとの連携
- Pixiv、Skeb等の日本サービスとの連携
